AUTH

Base URL: http://127.0.0.1:8000/auth/
Is responsible for managing authentication and authorization within the application. It provides mechanisms to issue, refresh, and verify authentication tokens, ensuring secure access to protected API functionalities. All data exchanged through this endpoint should be sent in JSON format, and requests must be made over HTTPS to maintain information security.

API

Base URL: http://127.0.0.1:8000/api/
Provides access to various functionalities through the routes /clients, /contacts, /sales, and /tasks. All routes are protected and require authentication via valid access tokens for any interaction. Additionally, certain routes or specific actions within them may be restricted based on the permissions associated with the authenticated user, ensuring that only authorized users can access or modify sensitive data.
Some fields in the request body act as foreign keys, meaning their associated records must be created first before making a request. For example category, client, position and tag fields must reference an existing instance. Each foreign key has its own dedicated route in the API, with full support for all HTTP methods (GET, POST, PUT, DELETE), enabling complete management of these related entities.

Sales

Sale instance.closed cannot be edited.

To close a sale put status field to 'done' or 'cancelled'.

The API includes a dedicated route, /sale-history/, which is accessible only via the GET and is Admin-only. This endpoint provides a detailed history of all sales. Since it is a read-only route designed for administrative oversight, it ensures secure and controlled access to sensitive sales data.

Filters

The API supports the use of query parameters to enhance and customize the results of requests.

search: Use this to perform a keyword-based search across fields specified for the endpoint. This is useful for quickly finding records matching certain criteria.
ordering: Use this to sort results based on one or more fields. Prefix a field with a minus sign (-) to sort in descending order (e.g., ?ordering=-created_at)
Specific field names:: Query parameters can also match specific field names, allowing you to filter results directly based on those fields.

/clients

  • ?category_id

/contacts

  • ?client_id
  • ?position_id
  • ?email

/sales

  • ?client_id
  • ?user_id
  • ?status

/tasks

  • ?status
  • ?user_id
  • ?tags__id (list)
